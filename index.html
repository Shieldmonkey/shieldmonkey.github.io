The "Static" Philosophy

不便さと引き換えに手に入れる、絶対的な安心。
Shieldmonkeyは「何もしない」ことであなたを守ります。

Zero Network Activity

拡張機能自体が外部との通信を一切行わないため、スクリプトの自動更新やクラウドバックアップも存在しません。この制約はCSP (connect-src 'self') によって物理的に担保されています。

No Silent Updates No Cloud Backup

Supply Chain Defense

NPMのベストプラクティスを徹底。npm ci による完全なバージョン固定と、ignore-scripts による実行防止で、ビルド環境の安全性を確保します。

Verified Dependencies

Verifiable Trust

セキュリティツールがブラックボックスであってはなりません。ビルド後のコードに対しても意図的にMinify（圧縮）を行わず、Source Mapsも同梱することで、誰もが検証可能な透明性を提供します。

How to Install

現在はパブリックベータ版として、手動インストールのみを提供しています。

1

GitHub Releasesからダウンロード

ページ下部のリンクから、最新の shieldmonkey-vX.X.X.zip をダウンロードし、解凍します。

2

デベロッパーモードを有効化

Chrome (またはEdge/Brave) の拡張機能管理ページ (chrome://extensions) を開き、右上の「デベロッパーモード」スイッチをONにします。

3

パッケージ化されていない拡張機能を読み込む

「パッケージ化されていない拡張機能を読み込む」ボタンをクリックし、解凍したフォルダ（manifest.jsonが含まれているフォルダ）を選択してください。

4

ユーザースクリプトを許可する

インストールされた拡張機能の詳細ページを開き、「ユーザースクリプトを許可する」のスイッチをONにしてください。
※この設定を行わないとスクリプトが動作しません。

Frequently Asked Questions

Q. なぜ手動インストールが推奨されるのですか？

Chrome Web Storeなどは「全自動更新」が基本です。しかし、セキュリティツールには「勝手にコードが変わらないこと」が求められます。Shieldmonkeyはユーザーがコードの変更点を確認し、意思を持ってアップデートすることを前提に設計されています。

Q. 利便性は下がりますか？

はい。自動更新やクラウド同期はありません。データのバックアップも手動のエクスポートが必要です。私たちは「便利さ」のために「安全」をトレードオフにすることをやめました。

Q. コードが読めるメリットは何ですか？

拡張機能があなたのブラウザ上で何をしているか、いつでも検証可能です。配布コードがMinify（圧縮）されていないため、ブラウザのデバッガーだけで「意図しない挙動」がないか容易に監査できます。

Q. ユーザースクリプト自体は安全ですか？

いいえ。Shieldmonkeyは「マネージャー」としての安全性を担保しますが、あなたがインストールする「スクリプト本体」の安全性までは保証できません。必ず信頼できるソースからスクリプトを取得し、内容を確認してから実行してください。
